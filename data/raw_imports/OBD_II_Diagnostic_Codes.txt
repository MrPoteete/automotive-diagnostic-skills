Part 2: Standardized DTC Failure Type Bytes (Suffix Codes)
These suffix codes (the sixth and seventh characters of an extended DTC) provide the specific electrical or logical failure mode as standardized by the SAE J2012/ISO 15031-6 documents. They are used to refine the diagnosis of the primary DTC (P0xxx, C0xxx, B0xxx, U0xxx).
| Failure Type Byte (Hex) | Description of Failure Mode |
|---|---|
| 00 | No Sub Type Information (Default or General Electrical Failure) |
| 01 | General Electrical Failure |
| 02 | General Signal Failure (General rationality or data fault) |
| 03 | Voltage Below Threshold (Circuit voltage is too low) |
| 04 | Voltage Above Threshold (Circuit voltage is too high) |
| 05 | Current Below Threshold (Circuit current is too low) |
| 06 | Current Above Threshold (Circuit current is too high) |
| 07 | Mechanical Failure |
| 08 | Bus Signal / Message Failure |
| 09 | Component Protection Activation |
| 0A | Rate of Change Below Threshold |
| 0B | Rate of Change Above Threshold |
| 0C | Signal Intermittent / Erratic / Incorrect |
| 0D | Calibration / Learning Not Performed |
| 0E | System Shutdown Due to Fault |
| 0F | Component / System Intermittent |
| 11 | Circuit Shorted to Ground |
| 12 | Circuit Shorted to Battery (or B+) |
| 13 | Circuit Open |
| 14 | Plausibility Fault / Signal Rationality Error |
| 15 | Reserved by SAE J2012 |
| 16 | Reserved by SAE J2012 |
| 17 | Reserved by SAE J2012 |
| 18 | Reserved by SAE J2012 |
